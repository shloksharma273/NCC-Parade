# Test Data

Test inputs and where drill outputs land.

## a) Testing data set — `dataset/`

Sample drill videos used for CLI runs and manual testing:

```
dataset/
├── front_salute.mp4
├── side_salute.mp4
└── WhatsApp Video 2026-06-26 at *.mp4
```

Run a detector against one of these:

```bash
python main.py --drill salute    --input test_data/dataset/front_salute.mp4
python main.py --drill kadam_tal --input test_data/dataset/your_video.mp4
```

## b) Testing data reports

Reports and analysis artifacts are **generated by the backend** and stored under
`backend/` (kept there so the running server can serve and index them via the DB):

| Location | Contents |
|----------|----------|
| `backend/reports/` | Per-session report JSON (`DRILL-*.json`) |
| `backend/media/reports/` | Rendered PDF reports |
| `backend/media/frames/` | Key / peak frame thumbnails |
| `backend/media/raw/` | Raw recorded drill videos |
| `backend/ml_output/<session>/` | Full ML output: annotated frames, `results.json`, `results.csv`, PDF |
| `backend/database/drill_server.db` | SQLite session/report index |

CLI runs (via `main.py`) write to the root `output/` directory instead.
